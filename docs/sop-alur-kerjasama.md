# SOP Alur Kerja Sama Layanan Copy Trading (EA)

## Tujuan
Menjelaskan alur kerja sama dari tahap edukasi pengguna hingga layanan berjalan, termasuk pengelolaan PKS dan keamanan data akses.

## Ruang Lingkup
Berlaku untuk seluruh calon pengguna dan tim admin/teknis layanan EA.

## Peran
- Admin: komunikasi, pengiriman dokumen, koordinasi tanda tangan.
- Tim teknis: setup EA, VPS, dan verifikasi koneksi.
- Pengguna: menyiapkan akun, menyetujui PKS, dan memberikan data akses trading.

---

## 1. Pra-Onboarding (Edukasi & Kecocokan)
1. Pengguna membaca landing page, Syarat & Ketentuan, dan Ringkasan PKS.
2. Pengguna menghubungi admin melalui WhatsApp.
3. Admin menjelaskan:
   - Model layanan (copy trading EA).
   - Akses trading saja (tanpa akses deposit/withdraw).
   - Risiko trading dan tidak ada jaminan profit.
   - Skema bagi hasil dan settlement bulanan.
4. Admin melakukan verifikasi kecocokan singkat:
   - Paham risiko.
   - Mindset jangka panjang.
   - Modal siap terekspos risiko pasar.

## 2. Pengiriman Dokumen PKS
1. Admin menyiapkan PKS lengkap (PDF) dengan data penyedia.
2. Admin meminta data calon pengguna (nama, alamat, identitas) untuk pengisian PKS.
3. Admin mengirim PKS lengkap untuk ditinjau pengguna.

## 3. Tanda Tangan PKS
1. Metode utama: e-signature (DocuSign/Adobe/Dropbox Sign).
2. Metode alternatif: tanda tangan manual pada PDF.
3. Setelah kedua pihak menandatangani, admin mengirim salinan PKS final kepada pengguna.

## 4. Pengumpulan Data Akses Trading
1. Pengguna menyiapkan:
   - Nomor akun trading.
   - Server akun.
   - Password trading/investor (bukan password utama broker).
2. Admin mengingatkan pengguna untuk tidak mengirim password utama broker.
3. Pengguna mengirim data melalui channel privat (WA/email) yang disepakati.

## 5. Setup Teknis
1. Tim teknis menjalankan EA di VPS yang disepakati.
2. Tim teknis memastikan:
   - Koneksi aktif.
   - Parameter risiko sesuai.
   - EA berjalan stabil.
3. Admin memberi tahu pengguna bahwa setup selesai dan meminta konfirmasi.

## 6. Aktivasi & Monitoring
1. Pengguna memantau laporan akun secara mandiri.
2. Admin memberikan update berkala (opsional) jika dibutuhkan.
3. Settlement dilakukan di akhir bulan:
   - Profit dihitung.
   - Bagi hasil 60% user / 40% provider.
   - Jika tidak profit, tidak ada fee.

## 7. Penghentian Layanan
1. Pengguna dapat menghentikan layanan kapan saja dengan mencabut akses trading.
2. Admin mengonfirmasi penghentian dan menutup proses teknis.
3. Tidak ada kewajiban lanjutan selain settlement yang sedang berjalan (jika ada).

---

## Catatan Keamanan
- Password trading/investor hanya untuk eksekusi, bukan kontrol dana.
- Password utama broker tidak boleh dibagikan.
- Sarankan pengguna mengganti password trading setelah setup bila diperlukan.

## Dokumentasi
- Simpan PKS final pada folder arsip internal.
- Simpan riwayat komunikasi penting (ringkasan) untuk audit internal.
